# Roadmap Auth / Onboarding Note

## Future addition to roadmap
Add an optional **Continue with Google** sign-in path alongside existing email/password authentication.

## Why
- reduces signup friction
- modernizes onboarding
- helps users start faster
- should not replace local SubnetOps account control

## Positioning
This is a future auth/onboarding improvement and should be tracked in the roadmap, but it is not the current priority over design-engine, topology, validation, and diagram work.
